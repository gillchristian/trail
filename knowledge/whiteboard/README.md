# whiteboard/ — cross-system (shared tier)

Discussions in flight — explored but not committed to action. Not a backlog:
entries record *what we considered* so future sessions don't reopen settled
ground. This is the **cross-system** whiteboard (root manifest, Layout): topics
spanning systems or the shared framework. Per-system topics belong in that
system's own whiteboard. Edited only via `MONO-` tasks (Shared-tier discipline).

One file per topic: `<topic-slug>.md` with Status line (open / resolved /
tabled), Context, Options considered, Where we landed, Follow-ups. When a topic
resolves: promote to an ADR, a backlog task, or a manifest/framework change —
and update the Status line; keep the entry (the recorded thinking is the value).
